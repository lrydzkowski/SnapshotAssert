# Proposal: revert-empty-value-whitespace

## Why

The D7 decision from `2026-07-12-fix-code-review-findings` made empty string values render with no separator space after the colon (`ApiKey:` instead of `ApiKey: `). This breaks every existing `.verified.txt` snapshot that contains an empty string value — including snapshots generated by real Verify 31.x in consumer projects migrating to SnapshotAssert — and silently violates the library's core promise in `snapshot-serialization`: byte-for-byte compatibility with Verify 31.x. Restoring Verify-compatible rendering removes the migration break for all consumers.

## What Changes

- **BREAKING** (relative to current post-D7 behavior): empty string property values render as `Name: ` again — colon, separator space, empty value — matching Verify 31.x. Any snapshot approved with the post-D7 `Name:` form must be re-approved.
- Empty string array elements render with the normal newline and indentation again (an indented line holding only indentation, plus the following element's separator comma where applicable), matching Verify 31.x.
- The `EmptyValuesNeverProduceLinesEndingInWhitespace` guarantee is withdrawn: rendered output may contain lines ending in whitespace when values are empty (or end in whitespace). The known trade-off — editors configured to trim trailing whitespace can corrupt `.verified.txt` files — is accepted in exchange for Verify compatibility.
- The other half of D7 is **kept**: property names containing `\n`/`\r` remain escaped to a single line, and the top-level `emptyString` sentinel for `Verify(string.Empty)` is unaffected.
- Writer, walker, and any golden tests locking the post-D7 rendering are updated to the Verify-compatible expectations.

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `snapshot-serialization`: the "Whitespace-safe output format" requirement is removed and replaced by (a) explicit empty-string scenarios under "Verify-compatible default text format" restoring the trailing-space rendering, and (b) a narrower "Single-line property names" requirement retaining only the name-escaping behavior.

## Impact

- `src/SnapshotAssert/Writing/VerifyTextWriter.cs`: `WriteRaw`/`BeginValue` lose the `isEmptyValue` special case.
- `tests/SnapshotAssert.Tests/Writing/VerifyTextWriterTests.cs`: four empty-value tests updated or removed.
- `tests/SnapshotAssert.Tests/Serialization/ObjectWalkerTests.cs`: `EmptyStringRendersWithoutTrailingSpace` updated.
- Repo-owned `.verified.txt` files (unit tests and `SnapshotAssert.PackageConsumer`) re-checked and re-approved where they contain empty values.
- `openspec/specs/snapshot-serialization/spec.md`: requirement restructuring per the delta spec.
- Consumers who approved snapshots against the post-D7 format must re-approve; consumers with Verify-era snapshots (the motivating case) start passing without edits.
